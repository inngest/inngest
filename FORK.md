# Fork management
## Git Strategy

### Keep a clean upstream tracking branch

```
git remote add upstream <original-repo-url>
git fetch upstream
git checkout -b upstream-main upstream/main
```

### Rebase your work on top of upstream regularly

```
git fetch upstream
git rebase upstream/main
```

### Code Organization

1. Prefer new files over modifying existing ones:
   Changes in existing files cause the most rebase
   conflicts. Where possible, add your customizations in new files that the existing code can
   import/call.

1. Use extension points
   Hook into interfaces, plugin systems, or configuration — don't patch core logic inline.

1. Group all fork-specific commits"
   Keep NCX-specific commits logically grouped and clearly labeled (e.g., [ncx] add inference billing
   hook). This makes them easy to identify and re-apply after a rebase conflict.

### Practical Conventions

1. Keep a FORK.md or similar file
   Document every file you've changed and why. When rebasing after a big upstream update, this is your
   checklist.

1. Avoid reformatting upstream files:
   Formatting-only changes create noise in diffs and conflict with upstream formatting changes.

1. Pin your fork commits at the tip:
   When your branch looks like:
   upstream commits → upstream commits → [ncx] change A → [ncx] change B
   ...rebasing is mechanical. Interleaved commits make it painful.

1. Use build tags or feature flags for invasive changes
   If you must modify a hot path, wrap it in a flag so the upstream behavior is preserved by default and
   your change is opt-in.

### Conflict Reduction

- Prefer composition over modification — wrap types/functions rather than editing them
- Avoid changing function signatures upstream relies on
- Watch the upstream changelog/PRs before rebasing to anticipate breaking changes

## Ignore: Git skip-worktree Files

because they are regenerated on every build. To disaable tracking:
`git update-index --skip-worktree ui/apps/dev-server-ui/src/store/generated-types.ts`
and to re-enable:
`git update-index --no-skip-worktree <file>`
